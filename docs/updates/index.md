Use this section to distinguish delivered capability from ongoing learning and future candidates.

## Four different questions

| Question | Read |
|---|---|
| What is supported now? | [Current state](https://nevitonsantana.github.io/adaptive-skills/updates/current-state/) |
| What changed for consumers? | [Changelog](https://nevitonsantana.github.io/adaptive-skills/updates/changelog/) |
| How do skills evolve safely? | [Evolution layer](https://nevitonsantana.github.io/adaptive-skills/evolution-layer/) |
| What may happen later? | [Efficiency Layer roadmap](https://nevitonsantana.github.io/adaptive-skills/efficiency-layer-roadmap/) |

A roadmap item is not delivered capability. An observation is not an approved proposal. A proposal is not a merged skill change.

## Governed evolution

The [Evolution layer](https://nevitonsantana.github.io/adaptive-skills/evolution-layer/) describes how observations, proposals, reviews, protected surfaces, and validation interact.

Supporting references include [Optimization boundaries](https://nevitonsantana.github.io/adaptive-skills/evolution/optimization-boundaries/), [Skill evolution experiments](https://nevitonsantana.github.io/adaptive-skills/evolution/skill-evolution-experiments/), [Validation case guidelines](https://nevitonsantana.github.io/adaptive-skills/evolution/validation-case-guidelines/), and [Recommended telemetry](https://nevitonsantana.github.io/adaptive-skills/telemetry/).
