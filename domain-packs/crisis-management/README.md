# Crisis Management

> **Example domain pack — not the canonical domain pack.** Adaptive Skills is domain-agnostic (see [ADR-002](../../docs/adr/ADR-002-domain-agnosticism.md)). This pack is the first concrete instance of the domain-pack pattern, drawn from the [Crisis Monitor field case](../../docs/crisis-monitor-case-study.md). Other domain packs across other domains are expected; this one is *one example*, not the reference.

Domain pack for crisis-management work. These skills are intentionally domain-specific and should not be treated as generic product skills.

## Current skills

crisis-analyst, monitoring-architect, cris-voice

## How to use this pack

This pack is useful when:
- The consumer project is a crisis monitoring / incident-response product.
- You want a worked, concrete example of how a domain pack is structured before authoring your own.

It is **not** useful when:
- You expect the canonical "domain pack template" — there is no canonical pack; see [ADR-002](../../docs/adr/ADR-002-domain-agnosticism.md).
- Your domain is unrelated to crisis monitoring and you would have to translate the assumptions.

For the skill model and the rules that apply to every domain pack, see [`docs/skill-model.md`](../../docs/skill-model.md).
