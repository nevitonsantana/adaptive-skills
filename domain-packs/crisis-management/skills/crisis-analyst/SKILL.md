# Overview

Use this skill when the job is to read a crisis situation coherently rather than simply summarizing mentions or coverage volume.

# When to Use

- briefing review
- narrative reading
- executive crisis status
- risk and escalation interpretation

# When NOT to Use

- generic reporting with no crisis lens
- monitoring setup work

# Core Moves

1. Define the actual crisis theme.
2. Separate new signal from noise and echo.
3. Name the stage of the situation.
4. State impact, acceleration risk, and near-term watchpoints.

# Optional Modules

- **Coverage gap** — Name what evidence is missing before the reading becomes overconfident.
- **Stakeholder map** — List who is most exposed or likely to react next.
- **Response sensitivity** — Flag whether the reading will likely demand institutional or spokesperson handling.

# Activation Triggers

- Use coverage gap when the evidence base is thin or uneven.
- Use stakeholder map when actors matter more than pure narrative volume.
- Use response sensitivity when the reading may trigger external messaging.

# Expected Output

- crisis reading
- stage statement
- main risks
- watchpoints

# Verification

- Signal and noise were distinguished.
- The stage is defensible from evidence.
- The output ends with consequence and next watchpoint.

# Handoff Signals

- Institutional response design is now needed.
- Monitoring setup or public narrative work becomes the next dominant layer.

# Pairs Well With

- `monitoring-architect`
- `cris-voice`
- `communication`

# Anti-patterns

- Treating mention volume as crisis severity by itself.
- Escalating tone without proportional evidence.
- Calling repetition a new fact.
